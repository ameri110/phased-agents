# Examples

Case studies of `phased-agents` applied to real projects. Useful for
seeing what the methodology looks like in practice across domains.

## Adding a case study

A good case study is short and concrete:

- **Domain** (algorithms / API / UI / mobile / infra / data)
- **Project shape** (size, timeline, team)
- **What the phases looked like** (just titles + gates)
- **A couple of notable wins** the methodology produced
  (a bug caught by the reviewer, a recalibration done via ADR, a
  cross-session handoff that worked cleanly)
- **A couple of notable failures** and what was learned
  (drift episodes, gates that turned out wrong, etc.)

Submit as `examples/<project-slug>.md`. Keep it under 500 lines. The
goal is to help readers see whether this methodology fits their
situation, not to ship a complete project history.
